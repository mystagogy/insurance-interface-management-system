# 보험 외부 인터페이스 운영관리 시스템 프로젝트 기획안

## 1. 프로젝트 개요

- 프로젝트명: 보험 외부 인터페이스 운영관리 백엔드 프로토타입
- 작성일: 2026-04-25
- 목적: 보험 업무 자체(가입/청약/지급) 구현이 아니라, 외부 보험 API 연동 운영을 안정적으로 조회/추적/관리하는 운영자 시스템 구축

### 배경

외부 Open API 기반 통계 연동은 장애 원인 파악, 호출 이력 추적, 재호출 판단이 어렵기 쉽다. 본 프로젝트는 운영자가 인터페이스 품질을 관리할 수 있도록 호출 상태와 이력을 일관되게 관리하는 데 집중한다.

## 2. 목표와 비목표

### 목표 (Goals)

- 외부 보험 API 3종 통합 조회
- API 호출 이력 저장/검색
- 성공/실패 상태 관리
- 실패 건 재처리 가능한 데이터 구조 확보
- 운영자 대시보드 제공

### 비목표 (Non-goals)

- 실제 보험 가입/청약/지급 업무 처리
- 실시간 결제/거래 엔진
- 복잡한 본인인증/전자서명 체계
- 대규모 분산 인프라 (멀티 리전, 이벤트 버스 중심)

## 3. 이해관계자

- 운영자(1차 사용자): 대시보드 확인, 조회 요청, 장애 이력 점검
- 개발자/유지보수자: 외부 API 스펙 변경 대응, 장애 원인 분석
- 평가자(과제 리뷰어): 재현 가능성, 아키텍처 준수, 운영성 확인

## 4. 문제 정의와 해결 전략

### 현재 문제

- API마다 응답 스키마가 달라 조회/가공 로직이 분산됨
- 실패/예외가 호출 이력과 분리되어 장애 추적성이 낮아짐
- 단순 조회 요청도 감사 이력(audit)으로 남기기 어려움

### 해결 전략

- `client` 계층에서 API별 스펙 차이 흡수
- `service` 계층에서 상태 전이와 호출 흐름 통제
- `api_call_history` + `error_log`를 분리 저장해 운영 추적성 강화
- `sync` 옵션 기반으로 저장 데이터 조회 / 외부 동기화 조회를 분리 제공

## 5. 범위 정의 (Scope)

### 포함 범위 (In Scope)

- 세션 기반 로그인
- 보험 통계 3종 조회 API
- 대시보드 요약 API(일일 총 호출/성공/실패)
- 호출 이력 조회 API
- Swagger 문서 노출
- Docker Compose 기반 로컬 실행

### 제외/향후 범위 (Out of Scope for Current Submission)

- 실패 건 자동 스케줄 재시도
- 수동 재처리 전용 API 엔드포인트
- 운영자 계정 자동 시드(현재는 DB 수동 생성)
- 고급 모니터링(알람, 메트릭 대시보드 연동)

## 6. 기능 요구사항

### FR-1. 인증/인가

- 로그인 API를 통해 세션 발급
- 인증되지 않은 요청은 차단
- Swagger, 로그인 페이지 등 일부 공개 경로 허용

### FR-2. 외부 통계 조회

- 자동차보험 계약 통계 조회
- 생명보험 가입 통계 조회
- 실손보험 가입 통계 조회
- `sync=true`면 외부 API 동기화 후 조회, `sync=false`면 저장 데이터 조회

### FR-3. 호출 이력 관리

- API 요청 시 PENDING 이력 생성
- 성공 시 SUCCESS, 실패 시 FAIL 상태 기록
- 요청 파라미터, 처리시간, 상태코드, 응답요약 저장

### FR-4. 장애 로그 관리

- 실패 시 ErrorType(TIMEOUT/CONNECTION/BAD_RESPONSE/UNKNOWN) 분류
- 에러 메시지 및 발생 시각 저장

### FR-5. 운영 대시보드

- 당일 호출 총 건수
- 당일 성공 건수
- 당일 실패 건수

## 7. 비기능 요구사항

- 보안: Session 기반 인증, 비밀번호 해시 저장(BCrypt)
- 운영성: 호출 이력과 에러 로그를 통한 추적 가능성 우선
- 유지보수성: Controller-Service-Repository 레이어 분리
- 확장성: API 추가 시 client/service/repository 모듈 단위 확장
- 재현성: `.env.example`, Docker Compose, README 기반 환경 복제

## 8. 성공 지표 (KPI)
